# Contributing to Sanskrit Search

Thank you for your interest in contributing! Here are some guidelines.

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
4. **Test your changes** thoroughly
5. **Commit with clear messages**: `git commit -m "Add: feature description"`
6. **Push to your fork**: `git push origin feature/your-feature-name`
7. **Create a Pull Request**

## Commit Message Guidelines

Use clear, descriptive commit messages:
- `Add:` for new features
- `Fix:` for bug fixes
- `Update:` for improvements to existing features
- `Docs:` for documentation changes
- `Test:` for adding tests
- `Refactor:` for code restructuring

## Code Style

- Use meaningful variable names
- Add comments for complex logic
- Follow existing code formatting
- Keep functions focused and modular

## Testing

- Test your changes in both browser and Node.js environments
- Add test cases for new features
- Ensure existing tests still pass

## Documentation

- Update README.md if you add new features
- Add JSDoc comments to new functions
- Update CHANGELOG.md

## Questions?

Feel free to open an issue for discussion before starting work on major changes.
